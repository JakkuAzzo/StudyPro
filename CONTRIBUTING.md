# Contributing to StudyPro

StudyPro welcomes bug fixes, accessibility improvements, study modes, documentation, translations, and original or openly licensed study content.

1. Create a focused branch from your fork.
2. Run `npm install`, `npm test`, and `npm run build`.
3. Exercise changed flows at desktop and mobile widths.
4. Explain the change and verification in your pull request.

Only submit content you wrote or may legally redistribute. Add the source and licence for public-sector or open material. Do not copy the licensed DVSA revision bank, official hazard clips, or commercial question banks merely because they are free to view.

Keep the app static and usable without accounts, analytics, API keys, or a backend. Preserve keyboard access, semantic labels, visible focus, and reduced-motion support. See [docs/CONTENT-GUIDE.md](docs/CONTENT-GUIDE.md) for the JSON and media schemas.

Code contributions are licensed under MIT. Content retains the licence stated in the pull request and source manifest.
